# 장소 리뷰 및 별점 기능 설계

## 목표

여행지 상세 화면에서 로그인 사용자가 1~5점의 필수 별점과 선택적인 리뷰 내용을 등록하고, 자신의 리뷰를 수정하거나 삭제할 수 있도록 한다. 장소 목록과 상세 화면에는 실제 활성 리뷰 데이터로 계산한 평균 별점과 리뷰 수를 표시한다.

## 사용자 경험

- 모든 사용자는 장소의 평균 별점, 리뷰 수, 최신순 리뷰 목록을 조회할 수 있다.
- 로그인 사용자는 별 5개 중 하나를 선택해 별점을 입력한다.
- 별점은 필수이며 리뷰 내용은 비워둘 수 있다.
- 별점을 선택하지 않은 상태에서는 등록 버튼을 비활성화하고 안내 문구를 표시한다.
- 장소당 사용자 리뷰는 하나만 허용한다.
- 사용자가 이미 리뷰를 작성했다면 작성 영역에 기존 별점과 내용을 채우고 수정 모드로 표시한다.
- 사용자는 자신의 리뷰만 수정하거나 삭제할 수 있다.
- 리뷰 삭제 시 확인 후 `PLACE_REVIEW` 행을 실제로 삭제한다.
- 비로그인 사용자가 작성 영역을 사용하려 하면 로그인 필요 안내를 표시한다.

## 프론트엔드 구조

### 리뷰 API 모듈

`src/entities/review/api/reviewApi.ts`에서 다음 요청과 타입을 관리한다.

- 장소 리뷰 목록 및 요약 조회
- 내 리뷰 등록
- 내 리뷰 수정
- 내 리뷰 삭제
- JWT가 필요한 요청의 `Authorization: Bearer <token>` 헤더 처리

### 장소 상세 화면

`PlaceDetailPage.vue`의 현재 로컬 목업 리뷰 배열을 서버 데이터로 교체한다.

- 리뷰 조회 상태, 로딩 상태, 오류 메시지
- 선택 별점 `1~5`
- 선택적인 리뷰 내용
- 현재 로그인 사용자의 리뷰 여부
- 등록·수정·삭제 처리 상태

별점 입력은 클릭 가능한 별 5개로 제공한다. 마우스 호버 시 예상 선택 범위를 보여주고, 클릭 시 선택값을 유지한다. 키보드 사용자를 위해 각 별 버튼에 접근 가능한 이름을 제공한다.

리뷰 목록의 각 항목에는 작성자 닉네임, 별점, 내용, 작성·수정 시각을 표시한다. 내용이 없는 리뷰는 별점과 작성자 정보만 표시한다.

### 장소 평점 표시

백엔드 장소 응답의 `averageRating`, `reviewCount`를 프론트 `Place.rating`, `Place.reviewCount`로 변환한다. 리뷰 등록·수정·삭제 후 상세 리뷰 응답에 포함된 최신 요약값으로 즉시 화면을 갱신한다.

## 백엔드 구조

### API

- `GET /api/places/{placeId}/reviews`
  - 공개 API
  - 최신순 리뷰 목록, 평균 별점, 리뷰 수 반환
  - 유효한 JWT가 있으면 `myReview`도 반환
- `POST /api/places/{placeId}/reviews`
  - 로그인 필수
  - 별점 필수, 내용 선택
  - 이미 리뷰가 있으면 충돌 응답
- `PUT /api/places/{placeId}/reviews/me`
  - 로그인 필수
  - 자신의 기존 리뷰 수정
- `DELETE /api/places/{placeId}/reviews/me`
  - 로그인 필수
  - 자신의 리뷰 행을 실제 삭제

### 요청 검증

- `rating`: 필수, 정수, 1 이상 5 이하
- `content`: 선택, 앞뒤 공백 제거, 빈 문자열은 `NULL` 저장
- `content`: 최대 1,000자
- 존재하지 않는 장소는 404
- 인증되지 않은 변경 요청은 401
- 중복 등록은 409
- 수정·삭제할 내 리뷰가 없으면 404

### 데이터 접근

기존 `PLACE_REVIEW` 테이블을 그대로 사용한다.

- `(place_id, user_id)` 고유 키로 장소당 사용자 리뷰 하나를 보장한다.
- 리뷰 목록은 `APP_USER`와 조인해 작성자 닉네임을 반환한다.
- 평균은 `AVG(rating)`, 리뷰 수는 `COUNT(*)`로 계산한다.
- 기존 데이터 중 `status = 'ACTIVE'`인 행만 조회·집계한다.
- 삭제는 `DELETE FROM PLACE_REVIEW WHERE place_id = ? AND user_id = ?`로 처리한다.

### 장소 조회 연동

장소 목록과 단건 조회 쿼리에 리뷰 집계 서브쿼리를 `LEFT JOIN`한다.

- 리뷰가 없으면 평균 별점 `0`
- 리뷰가 없으면 리뷰 수 `0`
- 평균 별점은 소수점 첫째 자리까지 반환

## 응답 모델

리뷰 조회 응답은 다음 정보를 포함한다.

```json
{
  "averageRating": 4.5,
  "reviewCount": 2,
  "reviews": [
    {
      "reviewId": 10,
      "userId": 3,
      "authorNickname": "여행자",
      "rating": 5,
      "content": "사진 찍기 좋아요.",
      "createdAt": "2026-06-21T12:00:00",
      "updatedAt": "2026-06-21T12:00:00",
      "mine": true
    }
  ],
  "myReview": {
    "reviewId": 10,
    "rating": 5,
    "content": "사진 찍기 좋아요."
  }
}
```

로그인하지 않았거나 작성한 리뷰가 없으면 `myReview`는 `null`이다.

## 오류 처리

프론트는 조회 실패 시 리뷰 영역에 재시도 가능한 메시지를 표시한다. 등록·수정·삭제 실패 시 입력값을 유지하고 사용자에게 오류 메시지를 보여준다. 인증 만료로 401이 반환되면 로그인 필요 메시지를 표시한다.

## 테스트

### 백엔드

- 별점 범위 및 내용 길이 검증
- 리뷰 등록, 중복 등록 거부
- 자신의 리뷰 수정 및 삭제
- 다른 사용자 리뷰에 영향을 주지 않는지 확인
- 평균 별점과 리뷰 수 집계
- 내용 없는 별점 리뷰 저장
- 비로그인 변경 요청 거부
- 존재하지 않는 장소와 리뷰 처리

### 프론트엔드

- 별점 미선택 시 등록 불가
- 별점 선택과 키보드 접근성
- 내용 없이 등록 가능
- 기존 리뷰가 있으면 수정 모드 표시
- 등록·수정·삭제 후 요약과 목록 갱신
- 비로그인 사용자 안내
- API 실패 시 입력값 보존 및 오류 표시
